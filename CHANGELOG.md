# Changelog

## 1.4.2

- Activation restart перечитывает durable updater state перед restart и retry, поэтому старый экземпляр плагина не может воскресить уже очищенный marker.
- Автоматические activation restart ограничены тремя попытками вместо бесконечного цикла.
- При удалении или перезагрузке плагина принадлежащие текущему module generation activation timers отменяются.
- Ошибка установки больше не оставляет кнопку в вечном состоянии `installing`: candidate освобождается для повторной попытки, а Telegram показывает причину и дальнейшее действие.
- Metadata плагина снова синхронизирована с корневым `VERSION`.

## 1.4.1

- Исправлена рассинхронизация корневого `VERSION` и metadata плагина.
- Updater state теперь привязан к корню Cardinal, а legacy JSON из прежней рабочей директории мигрируется автоматически.
- В JSON сохраняются запущенная локальная версия, включённые функции updater и получатели startup-уведомления; повреждённая remote version больше не сбрасывает остальные настройки.
- При каждом запуске выполняется raw `VERSION` comparison после короткого startup-окна без дублирующей немедленной проверки.

## 1.4.0

- Проверка точного raw `VERSION` выполняется при запуске и каждый час; ошибки не останавливают scheduler.
- Новая версия сначала сохраняется как immutable candidate и предлагается всем текущим authorized users; первое валидное глобальное решение побеждает.
- Решения «сейчас», «через час», автоматический timeout и отказ сохраняются на диске и восстанавливаются после перезапуска.
- Перезапуск после replacement не зависит от Telegram: уведомления надёжно продолжаются уже новым процессом.
- Выключение updater отменяет ожидающий candidate, не прерывая уже начатую явную установку.
- Ручная проверка больше не устанавливает обновление без решения пользователя.

## 1.3.0

- Добавлена команда `/lmsearch название или слово`: поиск по части названия без учёта регистра, inline-меню найденных лотов и полный список прямых ссылок.

## 1.2.0

- Добавлен отдельный корневой `VERSION`; metadata плагина синхронизирована с ним.
- Updater сначала проверяет `VERSION` по immutable commit SHA и скачивает код только для более новой версии из того же commit.
- Автообновления включены по умолчанию для новых установок, сохраняют явный legacy opt-out, работают в фоне и больше не блокируют команды запросом согласия.
- Добавлены однократное startup-уведомление всем авторизованным пользователям и явная авторизация updater callback-ов.
- Первая автопроверка отложена до короткого окна opt-out; частично успешная startup-рассылка продолжается без дублей успешным получателям.
- После обновления рассылается уведомление и сохраняется автоматический перезапуск через час с кнопкой немедленного перезапуска.

## 1.1.0

- Добавлены отключаемые opt-in автообновления и ручная проверка через `/lots_update`.
- Обновления загружаются по точному GitHub commit SHA, валидируются и устанавливаются с резервной копией.
- Добавлены настройки обновлений в главное меню и обязательное согласие при первом использовании.
