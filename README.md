# LotsManager

Плагин для [**FunPay Cardinal — версия sidor0912**](https://github.com/sidor0912/FunPayCardinal), который помогает просматривать, изменять, импортировать, экспортировать и копировать лоты через Telegram-интерфейс.

> [!NOTE]
> **LotsManager — это vibecode-версия оригинального плагина Lots Copy Plugin** из Telegram-канала [FunPay Cardinal Plugins](https://t.me/fpc_plugins). Исходная идея и основа принадлежат авторам оригинального плагина; эта версия доработана с помощью ИИ и не является официальным релизом исходных авторов или FunPay Cardinal.

> [!WARNING]
> Плагин выполняет массовые и необратимые операции. Перед импортом или удалением создайте резервную копию и сначала проверьте работу на тестовом аккаунте.

## Возможности

- просмотр и редактирование лотов через inline-кнопки;
- включение, отключение и массовые операции;
- безопасный JSON-экспорт без `golden_key` и данных автовыдачи;
- импорт лотов с сохранением исходных ID;
- прямое копирование на другой аккаунт;
- история отключённых лотов и пользовательские теги.

## Требования и установка

- Python 3.10+;
- установленный и настроенный [FunPay Cardinal от sidor0912](https://github.com/sidor0912/FunPayCardinal);
- совместимые версии `FunPayAPI` и `pyTelegramBotAPI`, поставляемые вашим Cardinal.

Скопируйте `LotsManager.py` в каталог плагинов Cardinal и перезапустите Cardinal. Точная совместимость с конкретным форком Cardinal пока не зафиксирована — перед использованием массовых операций проверьте их на тестовом окружении.

## Безопасность

- Никогда не публикуйте `golden_key`, файлы из `storage/` или полные логи.
- JSON-экспорт всегда очищается от токенов и данных автовыдачи.
- Настройка `with_secrets` действует только при прямом копировании на второй аккаунт.
- Бот пытается удалить сообщение с `golden_key`, однако Telegram и инфраструктурные резервные копии могут сохранить его. Используйте приватный чат и после операции отзовите ключ при любых подозрениях.
- Экспортированные временные JSON-файлы удаляются после отправки.

Runtime-данные создаются в `storage/plugins/` и `storage/cache/`; они исключены из Git.

## Основные команды

- `/manage_menu` — главное меню;
- `/manage_lots` — управление лотами;
- `/lmsearch название или слово` — найти похожие лоты, открыть меню результатов и получить ссылки;
- `/cache_lots` — экспорт в JSON;
- `/create_lots` — импорт из JSON;
- `/copy_lots` — копирование на другой аккаунт;
- `/copy_with_secrets` — перенос автовыдачи при прямом копировании;
- `/lot_subcats` — настройка подкатегорий;
- `/lots_tags` — создание и обновление тегов.
- `/lots_update` — включение, отключение и ручная проверка обновлений.

## Обновления

На новой установке и при миграции без ранее сохранённого явного отказа автообновления включены по умолчанию и
не блокируют команды. Явный legacy opt-out сохраняется. Через 7 секунд
после инициализации каждый пользователь из `cardinal.telegram.authorized_users` получает однократное для версии
уведомление с возможностью глобально выключить автообновления или оставить текущую настройку. На первой обработке
версии фоновая проверка начинается ещё через 8 секунд, оставляя короткое окно для отказа. Успешные получатели
запоминаются, поэтому после частичной ошибки на следующем старте уведомление повторяется только для failed/new
пользователей. Запущенная локальная версия, включённые функции updater, выбор и прогресс рассылки сохраняются в
`storage/plugins/lots_manager_updater.json` относительно стабильного корня Cardinal и могут быть изменены командой
`/lots_update` или в меню настроек. Старый updater JSON из process working directory автоматически переносится при
первом запуске, если стабильного файла ещё нет.

Обновления загружаются только из публичного репозитория
[`voterol/FpcLotsManager`](https://github.com/voterol/FpcLotsManager). При запуске и затем каждый час updater строго
проверяет `https://raw.githubusercontent.com/voterol/FpcLotsManager/refs/heads/main/VERSION`. Для новой версии он
разрешает точный commit файла `VERSION` через GitHub API и повторно проверяет документ по SHA. При равенстве `LotsManager.py` не скачивается,
downgrade запрещён. Для новой версии код загружается из того же SHA, а metadata `VERSION` обязана совпасть с
корневым `VERSION`. Затем проверяются HTTPS-адрес, размер, синтаксис, `NAME` и `UUID`; создаётся резервная копия,
и только собственный `.py`-файл заменяется атомарно.

Новый immutable candidate предлагается всем текущим авторизованным пользователям кнопками «Обновить сейчас»,
«Обновить позже (через час)» и «Не обновлять». Первое валидное решение действует глобально; без решения через час
происходит автоматическая установка. После безопасной замены updater сохраняет durable notice и сразу перезапускает
Cardinal, не ожидая Telegram; уже новый процесс уведомляет всех зафиксированных получателей с сохранением прогресса.
Candidate, deadline и прогресс рассылки хранятся на диске и восстанавливаются после падения. Выключение updater
отменяет undecided/later candidate, но не прерывает уже начатую явным решением установку. Отказ относится только
к точному commit и не отключает будущие проверки. Ручная проверка только обнаруживает candidate и показывает prompt.
Activation restart перед каждой попыткой перечитывает durable state и ограничен тремя автоматическими попытками,
чтобы stale module instance или возвращающий управление restart handler не создавал бесконечный цикл.

Включение обновлений означает доверие владельцу указанного репозитория на публикацию исполняемого кода.
