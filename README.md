# LotsManager

Плагин для [**FunPay Cardinal — версия sidor0912**](https://github.com/sidor0912/FunPayCardinal), который помогает просматривать, изменять, импортировать, экспортировать и копировать лоты через Telegram-интерфейс.

> [!NOTE]
> **LotsManager — это vibecode-версия оригинального плагина Lots Copy Plugin** из Telegram-канала [FunPay Cardinal Plugins](https://t.me/fpc_plugins). Исходная идея и основа принадлежат авторам оригинального плагина; эта версия доработана с помощью ИИ и не является официальным релизом исходных авторов или FunPay Cardinal.

> [!WARNING]
> Плагин выполняет массовые и необратимые операции. Перед импортом или удалением создайте резервную копию и сначала проверьте работу на тестовом аккаунте.

## Возможности

- просмотр и редактирование лотов через inline-кнопки;
- включение, отключение и массовые операции;
- безопасный JSON-экспорт без `golden_key` и данных автовыдачи;
- импорт лотов с сохранением исходных ID;
- прямое копирование на другой аккаунт;
- история отключённых лотов и пользовательские теги.

## Требования и установка

- Python 3.10+;
- установленный и настроенный [FunPay Cardinal от sidor0912](https://github.com/sidor0912/FunPayCardinal);
- совместимые версии `FunPayAPI` и `pyTelegramBotAPI`, поставляемые вашим Cardinal.

Скопируйте `LotsManager.py` в каталог плагинов Cardinal и перезапустите Cardinal. Точная совместимость с конкретным форком Cardinal пока не зафиксирована — перед использованием массовых операций проверьте их на тестовом окружении.

## Безопасность

- Никогда не публикуйте `golden_key`, файлы из `storage/` или полные логи.
- JSON-экспорт всегда очищается от токенов и данных автовыдачи.
- Настройка `with_secrets` действует только при прямом копировании на второй аккаунт.
- Бот пытается удалить сообщение с `golden_key`, однако Telegram и инфраструктурные резервные копии могут сохранить его. Используйте приватный чат и после операции отзовите ключ при любых подозрениях.
- Экспортированные временные JSON-файлы удаляются после отправки.

Runtime-данные создаются в `storage/plugins/` и `storage/cache/`; они исключены из Git.

## Основные команды

- `/manage_menu` — главное меню;
- `/manage_lots` — управление лотами;
- `/cache_lots` — экспорт в JSON;
- `/create_lots` — импорт из JSON;
- `/copy_lots` — копирование на другой аккаунт;
- `/copy_with_secrets` — перенос автовыдачи при прямом копировании;
- `/lot_subcats` — настройка подкатегорий;
- `/lots_tags` — создание и обновление тегов.
- `/lots_update` — включение, отключение и ручная проверка обновлений.

## Обновления

При первом использовании плагин спрашивает, разрешить ли автоматические обновления. Выбор сохраняется в
`storage/plugins/lots_manager_updater.json` и может быть изменён командой `/lots_update` или в меню настроек.

Обновления загружаются только из публичного репозитория
[`voterol/users-voterol-fpc`](https://github.com/voterol/users-voterol-fpc) по неизменяемому commit SHA. Плагин
проверяет HTTPS-адрес, размер, синтаксис, `NAME`, `UUID` и версию файла, создаёт резервную копию и атомарно
заменяет только собственный `.py`-файл. Новая версия начинает работать только после перезапуска Cardinal.

Включение обновлений означает доверие владельцу указанного репозитория на публикацию исполняемого кода.

## Проверка

```bash
python3 -m compileall -q LotsManager.py
python3 -m unittest discover -s tests -v
```

Интеграционные проверки требуют запущенного Cardinal, тестового Telegram-бота и тестового FunPay-аккаунта.
